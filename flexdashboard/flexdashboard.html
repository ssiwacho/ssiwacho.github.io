Rmarkdown: Flexdashboard

สิวะโชติ ศรีสุทธิยากร

ภาควิชาวิจัยและจิตวิทยาการศึกษา คณะครุศาสตร์
จุฬาลงกรณ์มหาวิทยาลัย
13 May 2021

Rmarkdown เป็น package ที่มีความยืดหยุ่นสูงมากและสามารถประยุกต์ใช้ได้อย่างหลากหลาย ในบทเรียนนี้จะกล่าวถึงการสร้าง dashboard ด้วย package-flexdashboard รายละเอียดมีดังนี้

การติดตั้ง package

ก่อนใช้งานผู้ใช้สามารถดาวน์โหลดและติดตั้ง package ได้โดยพิมพ์คำสั่งดังนี้

# install.packages('flexdashboard')
library(flexdashboard)

Introduction

Dashboard เป็นรูปแบบของรายงานประเภทหนึ่ง มีวัตถุประสงค์หลักเพื่อนำเสนอสารสนเทศโดยสรุปของตัวชี้วัดที่สำคัญต่อการดำเนินงานขององค์กรหรือหน่วยงานต่าง ๆ ลักษณะของ dashboard จะเป็นชุดของทัศนภาพข้อมูลที่นำมาจัดเรียงให้อยู่ในหน้าเดียวกัน โดยทัศนภาพข้อมูลภายในชุดดังกล่าวอาจมีความแตกต่างกันได้ ทั้งนี้ขึ้นอยู่กับลักษณะข้อมูลและวัตถุประสงค์ของการนำเสนอ ซึ่งเป็นไปได้ทั้ง การนำเสนอค่าสถิติเพียงค่าเดียว แผนภาพ/แผนภูมิ แผนที่ ข้อความ หรือตาราง เป็นต้น

การใช้ dashboard เป็นวิธีการที่มีประสิทธิภาพที่ช่วยให้ผู้เกี่ยวข้องสามารถเข้าถึงสารสนเทศที่สำคัญได้เหมือน ๆ กันจากแหล่งข้อมูลเดียว

flexdashboard เป็น package หนึ่งของโปรแกรม R ที่ใช้ R markdown เป็นพื้นฐานสำหรับจัดวางทัศนภาพข้อมูลทให้มีหน้าตาแบบ dashboard จุดเด่นของการใช้ flexdashboard คือชุดคำสั่งที่สั้นและง่ายต่อการเขียน ซึ่งช่วยให้ผู้ใช้งานโฟกัสกับการออกแบบเนื้อหาหรือสารสนเทศที่ต้องการนำเสนอใน dashboard มากกว่าการเขียนคำสั่ง

ข้อดีอีกประการหนึ่งคือการใช้ flexdashboard ผู้ใช้จะสามารถใช้ engine ของ R เป็น backend ของ dashboard ได้ทั้งหมด จึงทำให้สามารถสกัดสารสนเทศจากข้อมูลได้อย่างลุ่มลึกพร้อม ๆ กับการสร้างทัศนภาพข้อมูลที่ต้องการได้อย่างมีประสิทธิภาพ นอกจากนี้ flexdashboard ยังรองรับการ interactive element อีกด้วย

Anatomy of flexdashboard

  • YAML header สำหรับ flexdashboard ให้กำหนด output type เป็นดังคำสั่งด้านล่าง
# ---
title: "Untitled"
output: 
  flexdashboard::flex_dashboard:
# ---
  • Layout การสร้าง dashboard คือการกำหนด layout เพื่อวางทัศนภาพข้อมูลลงในพื้นที่ที่กำหนด มีหลายวิธีการที่ใช้สำหรับกำหนดจุดวางทัศนภาพข้อมูลลงบน dashboard วิธีการง่ายที่สุดสามารถทำได้ดังนี้

กิจกรรม 1

ดาวน์โหลดไฟล์ act1.rmd จากนั้นบนหน้าต่าง R console ให้พิมพ์คำสั่งด้านล่าง แล้วจากนั้นสังเกตผลลัพธ์ที่ได้

rmarkdown::render("act1.rmd")

กิจกรรม 2

อีกวิธีการหนึ่งคือสามารถสร้าง dashboard ได้จาก Rstudio โดยคลิกเลือกที่ File>New File>RMarkdown แล้วเลือก From Template จากนั้นเลือก Flex Dashboard แล้วคลิก OK

การสร้าง dashboard จาก Rstudio สามารถทำได้โดยกดปุ่ม knit เหมือนกับการสร้างรายงานแบบ R Markdown ปกติ

Layout

  • ผู้ใช้สามารถกำหนดความกว้างของแต่ละ column ได้โดยกำหนด attribute data-width=xxx เพิ่มเติมด้านหลังการระบุ Column ดังรูปด้านบน

  • นอกจากนี้ผู้ใช่้ยังสามารถเปลี่ยนลำดับการจัดเรียงทัศนภาพข้อมูลใน dashboard ใหม่ จากค column เป็น row ได้ โดยกำหนด header ใหม่ดังนี้

# ---
title:"Untitled"
output:flexdashboard::flex_dashboard:orientation:rows
# ---

กิจกรรม 4

  • ลองปรับ header ตามคำสั่งข้างต้น จากนั้น render dashboard ขึ้นมา แล้วสังเกตผลลัพธ์ที่ได้ ในกรณีนี้หากต้องการกำหนดขนาดของ row แต่ละอันสามารถทำได้โดยกำหนด attribute data-height=xxx

  • นอกจากนี้ในหลายกรณีทัศนภาพข้อมูลที่สร้างขึ้นอาจมีขนาดใหญ่ไม่พอดีกับช่องใน dashboard กรณีนี้ผู้ใช้สามารถกำหนดให้แต่ละช่องเป็น scrollable box ได้โดยกำหนด header เพ่ิมเติมดังนี้
# ---
title:"Untitled"
output:flexdashboard::flex_dashboard:orientation:rows
vertical_layout:scroll
# ---

อย่างไรก็ตามการออกแบบ dashboard ลักษณะนี้ไม่ค่อยเหมาะสมนัก ถ้าไม่จำเป็นไม่ควรทำ

กิจกรรม 5

สร้างทัศนภาพข้อมูลลงในแต่ละช่องของ dashboard ให้เต็ม

Tabsets

ในทำนองเดียวกับ rmarkdown ผู้ใช้สามารถเพิ่ม tabset เพื่อบรรจุทัศนภาพข้อมูลหลาย ๆ ตัวภายใน box เดียวกันได้ โดยใช้ผู้อ่านเลือกดูตามความต้องการ โดยระบุ attrubute .tabset ลงบน row หรือ column ที่ต้องการสร้าง tabset ดังตัวอย่างคำสั่งด้านล่าง

Multiple pages

ผู้ใช้สามารถสร้าง dashboard ให้มีหลายหน้าต่างได้ โดยเขียนคำสั่งดังนี้

reference: (https://fontawesome.com/v4.7/icons/)

นอกจากการนำเสนอ multiple page ในลักษณะข้างต้นแล้ว ยังสามารถนำเสนอ multiple page ในลักษณะ drop down menu ได้ดังนี้

ผลลัพธ์จากชุดคำสั่งข้างต้นจะได้ dashboard ดังรูปต่อไปนี้

Setting page orientation

เราสามารถกำหนดให้แต่ละ page มีการจัดวาง layout แตกต่างกันได้ดังตัวอย่างต่อไปนี้

Story boards

นอกจากการจัดวางทัศนภาพข้อมูลตาม column และ row แล้ว flexdashboard ยังมี option การนำเสนอแบบ storyboard โดยสามารถนำเสนอทัศนภาพข้อมูลประกอบคำบรรยายไปตามลำดับที่ต้องการได้

Components

ส่วนประกอบใน flexdashboard มีด้วยกันหลายประเภท ดังนี้

  1. interactive JavaScript data visualization

  2. R graphical output (base, ggplot2, lattice, highcharter, …)

  3. Tabular data (with optional sorting, filtering, and paging)

  4. Value boxes

  5. Gauges

  6. Text annotations

  7. navigation bar

Values boxes

การสร้าง values boxes เป็นการรายงานค่าสถิติหรือตัวเลขเพียงตัวเดียว สำหรับนำเสนอสารสนเทศเพื่อแสดงสถานะหรือระดับของตัวบ่งชี้สำคัญ การสร้าง value boxes สามารถทำได้โดยใช้ฟังก์ชัน valuebox() ดังตัวอย่างต่อไปนี้

ฟังก์ชัน valuebox() มีอาร์กิวเมนท์ icon ใช้สำหรับกำหนดรูปแบบของ icon ที่ใช้ประกอบ value box และมีอาร์กิวเมนท์ color ที่ใช้สำหรับกำหนดสีเพื่อแสดงสถานะให้กับ value box โดยมีสีสำเร็จรูปให้เลือกใช้ได้แก่ primary, info, success, warning และ danger นอกจากนี้ผู้ใช้ยังสามารถกำหนดสีได้เองโดยใช้ได้ทั้งระบบสีแบบ RGB และ hex code

Gauges

gauges เป็นรูป meter ใช้ระบุระดับหรือสถานะของตัวบ่งชี้สำคัญในทำนองเดียวกับ value box ส่วนที่แตกต่างกันคือ gauges ให้สารสนเทศเกี่ยวกับขอบเขตของค่าตัวบ่งชี้ดังกล่าวด้วย การสร้าง gauges ใน flexdashboard สามารถทำได้โดยใช้ฟังก์ชัน gauge() ดังตัวอย่างต่อไปนี้

จากตัวอย่างข้างต้นจะเห็นว่า

  1. gauge มีอาร์กิวเมนท์ที่จำเป็นต้องระบุ 3 ตัวได้แก่ value, min และ max ซึ่งต้องระบุเป็นตัวเลข โดยจะเป็นตัวเลขอะไรก็ได้ นอก

  2. ผู้ใช้สามารถกำหนดสัญลักษณ์แสดงหน่วยข้อมูลเพิ่มเติมผ่านอาร์กิวเมนท์ symbol ดังตัวอย่างข้างต้นที่กำหนดสัญลักษณ์ % ไว้หลังตัวเลข 91

  3. สีของแท่งใน meter สามารถเปลี่ยนแปลงได้เองตาม cutoff ที่กำหนด การกำหนด cutoff ดังกล่าวสามารถกำหนดผ่านอาร์กิวเมนท์ gaugeSectors(success, warning, danger) ที่ให้กำหนดได้ 3 ระดับคือ success, warning และ danger ดังตัวอย่างข้างต้น

Text Annotations

การใส่ข้อความบรรยายใน dashboard อาจใส่ได้สองลักษณะ

  1. ใส่ข้อความบริเวณส่วนบนของ dashboard เรียกว่าเป็นส่วนนำหรือ introduction ของ dashboard

  2. ใส่ส่วนใสส่วนหนึ่งของ dashboard ข้อความที่ใส่เขียนด้วยภาษา Markdown และเป็นไปได้ทั้งข้อความธรรมดา รูปภาพ สมการทางคณิตศาสตร์ เป็นต้น นอกจากนี้ยังสามารถฝัง hyperlink ให้กับข้อความหรือวัตถุต่าง ๆ ได้อีกด้วย
